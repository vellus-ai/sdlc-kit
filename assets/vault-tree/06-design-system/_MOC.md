# Design System

> Tokens, components and UI patterns — the visual and interaction library of the product.

## Purpose

Here live the building blocks of the design system: tokens (colors, typography, spacing, radius), components (with props, variants, states, accessibility) and patterns (when to use, anti-patterns). NOT here: component code (lives in the frontend/design repository) nor binary assets (use the design repository).

## Subfolders

- `tokens/` — tokens named with value, usage and aliases.
- `components/` — components with API, variants and examples.
- `patterns/` — UI patterns and when to apply them.
