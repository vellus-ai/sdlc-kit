# Retrospectives

> Continuous learning: sprint retros, PR reviews, closed incidents and lessons learned.

## Purpose

This folder is the team's memory of what was learned: retros per cycle, PR reviews with AppSec/quality checklist, resolved incidents with timeline and lessons learned (5-whys). NOT here: live incidents (use Grafana/PagerDuty), runbooks (live where the team already maintains them).

## Subfolders

- `retros/` — one file per closed sprint (pre-populated by the `sprint close` hook).
- `reviews/` — PR reviews with AppSec and quality checklist.
- `incidents/` — resolved incidents + `<slug>-lessons.md` (generated by the `resolved` status hook).
