# Architecture

> Structural decisions, C4 models, cross-feature TRDs and API contracts — the technical HOW.

## Purpose

This folder records the system architecture at multiple levels: ADRs (individual decisions), C4 (context, container, component), TRDs (cross-feature technical requirements) and API contracts by type. NOT here: feature-level functional requirements (use `04-specs/<feature>/`) nor domain modeling (use `03-domain/`).

## Subfolders

- `adr/` — Architecture Decision Records numbered (`ADR-NNNN-<slug>.md`).
- `c4/` — C4 diagrams at context, container, and component levels.
- `trd/` — Technical Requirements Docs cross-feature (NFRs, SLIs/SLOs).
- `api/` — contracts by type (`rest/`, `grpc/`, `async/`, `webhook/`); created on-demand by the `spec` skill.
