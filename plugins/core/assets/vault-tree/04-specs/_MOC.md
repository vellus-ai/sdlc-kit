# Specs

> Spec-Driven Development: one folder per feature, with requirements (EARS), design and tasks.

## Purpose

Each feature has a subfolder `<feature>/` containing the canonical trio: `requirements.md` (EARS requirements, testable), `design.md` (feature architecture) and `tasks.md` (executable task list). NOT here: PRDs or epics (use `01-planning/`); stories live as bullets within epics, not as separate files.
