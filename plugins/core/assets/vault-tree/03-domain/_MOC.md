# Domain

> DDD modeling: bounded contexts, ubiquitous language, aggregates, events and context contracts.

## Purpose

This folder materializes the mental model of the business: the context map, the ubiquitous language glossary, the aggregates with their invariants, domain events and contracts (ACLs) between bounded contexts. NOT here: technical API contracts (use `02-architecture/api/`) nor implementation (lives in code).

## Subfolders

- `aggregates/` — one file per domain aggregate.
- `events/` — one file per domain event.
- `contracts/` — integration contracts between bounded contexts (ACL, OHS, etc.).
